Phone Controller — Android 디바이스 모니터링/제어 도구

최종 갱신 2026-05-28 · 대상 디바이스 SM-M446K (Snapdragon 888 / Android 15)

1. 개요

ADB 위에서 동작하는 Tkinter GUI. 폰에 작은 TCP 데몬을 올려 화면 미러링/시스템 모니터링/입력 제어/매크로/온디바이스 LLM 런타임 병목 분석을 한 곳에서 수행합니다. 시작은 작업관리자 + 화면 미러로 출발했고, 스펙 문서(android_llm_runtime_monitoring_request.md)를 받아 LLM 런타임 인스펙터까지 확장.

아키텍처

┌──────────────────────────────────────┐         ┌──────────────────────────┐
│  PC: gui.py (Tkinter)                │  TCP    │  Phone: nc -L :8889      │
│  - device picker                     │ <═════> │  fork handler.sh         │
│  - timeline / charts / detail views  │ adb fwd │  - input / dumpsys /     │
│  - artifacts session writer          │         │    /proc /sys parsing    │
│  - perfetto/simpleperf orchestration │         │  - screenrecord stream   │
└──────────────────────────────────────┘         └──────────────────────────┘

2. 구현된 기능 (카테고리별)

실시간 시스템 모니터링

  • CPU 전체%, 코어별 freq · MEM% · CPU 온도 · GPU 모델/busy/freq · Battery · Power(mW, root 환경)
  • 1.5초 폴링, 클라이언트 측 delta 계산
  • thermal zone 자동 스캔

프로세스/스레드 분석

  • foreground 앱 + 사용자앱 CPU/MEM 순위 (top -m 80, USER ~ /^u0_a/)
  • 우클릭 → threads: per-TID CPU%/runqueue WAIT% (proc/task/stat+schedstat)
  • 우클릭 → I/O+sched: /proc/io rchar/wchar/read_bytes 델타 + cpuset/cgroup/governor
  • 우클릭 → memory detail: dumpsys meminfo + smaps_rollup + PSS sparkline + 시작점 대비 Δ

화면 미러링

  • PNG 모드: screencap -p 폴링 (조정 가능)
  • H.264 모드: screenrecord stdout → TCP → PyAV 디코드 (~60fps)
  • 캔버스 클릭/드래그 → tap/swipe

제어 / 입력

  • HW key 버튼 (Home/Back/Power/Vol±/Menu/Enter)
  • 앱 런처 (Instagram/Browser/Settings/...)
  • PC 키보드 → 폰 패스스루 (한글은 ADBKeyboard IME)
  • raw shell 명령창

매크로

  • 녹화: TAP/SWIPE/KEY/TEXT 시퀀스 + 상대 timestamp
  • Push&Run: detached .sh 푸시, setsid 백그라운드 실행 → USB 분리해도 동작
  • loop 1/n/∞ 지원, "Stop all" 버튼
  • 매크로별 pidfile + 종료 시 자동 cleanup

안정성 (Stability 5탭)

  • logcat crash buffer
  • ANR (/data/anr + dropbox)
  • tombstones (/data/tombstones)
  • binder dump
  • activity processes

3. Jank / UI 디버깅 (Android Studio Profiler 스타일)

프로세스 우클릭 → "jank timeline" → 1180×540 윈도우. 

┌─ legend(좌) ── stage chips(중) ──────────────────── donut(우) ──────┐
│                                                                      │

│  UI │  ▮▮▮  ▮▮  ▮▮▮▮  ▮▮  ▮▮          (IntendedVsync→SyncQueued)   │
│  RT │     ▮▮     ▮▮▮     ▮▮              (SyncStart→FrameCompleted) │
│      sync-wait는 점선 커넥터로 연결                                  │
│  −5s ──────────────────────────────────────────────────── now        │
└──────────────────────────────────────────────────────────────────────┘
                                                       ┌─ detail panel ─┐
                                                       │ Frame #idx     │
                                                       │ 8단계 컬러 막대 │
                                                       │ dominant 강조  │
                                                       └────────────────┘

특징

  • 2-lane 시간축 — UI 스레드 / RenderThread 시간정렬
  • 색상 (AS 컨벤션): on-time slow (1–2× deadline) janky (≥2×) frozen (≥700ms)
  • vsync 데드라인 자동 — dumpsys display로 60/90/120Hz 감지해서 deadline_ms 동적 조정
  • hover: 막대 위에서 frame# · 총 ms · lane 즉시 표시
  • 클릭: 우측 패널에 8단계 분해 (input · animation · traversal · draw · sync wait · issue cmds · gpu work · gpu swap) + dominant 단계 + 해석 힌트
  • 도넛: 화면에 보이는 모든 프레임의 누적 stage 비율, 가운데 frame 개수, 아래 dominant %
  • Layer stats 버튼: SurfaceFlinger per-layer latency 히스토그램 (app_vsync→present)
  • Auto UIA on jank 체크박스 — janky frame(≥2× deadline) 감지 시 uiautomator dump 자동 실행(30s throttle), session_dir/uia/uia_<ts>_fc<n>_<dur>ms.xml 저장 + 활성 패키지/노드 수 표시
  • All Frames 버튼 — Android Studio "All Frames" 탭처럼 모든 frame을 표(#, t, Total ms, UI ms, RT ms, Status, Dominant stage)로. on-time/slow/janky/frozen 색상, "Follow newest" 토글로 자동 스크롤

3D view tree (Layout Inspector 스타일)

메인 toolbar "3D view" 버튼 → uiautomator dump 캡처 → view hierarchy를 depth 축으로 ortho 투영해서 회전 가능한 3D로 그림.

  • 좌측: 3D 캔버스. 드래그=회전, 휠=zoom, 클릭=선택(드래그 ≠ 클릭, 4px 이내만 클릭), 더블클릭=카메라 리셋 + autofit
  • 우측 상단: View hierarchy 트리(들여쓰기, 숨겨진 노드는 회색). 캔버스 클릭 ↔ 트리 선택 양방향 동기화
  • 우측 중간: Visible 체크박스 + Hide / Solo(선택+ancestors+descendants만 보이기) / Show all 버튼
  • 우측 하단: Properties — class, resource-id, package, text, content-desc, bounds(폭×높이), clickable/long-clickable/enabled/focused/scrollable/checkable/checked/selected/password/visible 전부
  • 선택된 노드는 흰색 두꺼운 outline + 진한 stipple로 강조

저장 위치: artifacts/_uia/uia_<ts>_3d.xml (세션 중이면 session_dir/uia/).

스레드 detail (per-TID 시계열)

프로세스 우클릭 → threads → 행 더블클릭 → 선택 TID의 시계열 detail 윈도우. 1Hz 폴링으로 ~5분 윈도우. CPU%(빨강) + runqueue WAIT%(노랑) 라인 차트, R/S/D/Z/T/I state 색띠 스트립, 마지막 wchan + voluntary/nonvoluntary context switches, kernel stack(있으면). stack은 root가 보통 필요하므로 못 읽으면 "(needs root)" 표시.

4. 시계열 차트 (메인 화면 하단)

1.5초마다 갱신, 600 샘플(15분) 롤링 버퍼.

  • 6개 시리즈 동시 표시 (각각 토글 가능): CPU MEM GPU TEMP GPU MHz POWER
  • 윈도우 토글: 1분 / 5분 / 15분
  • foreground 앱 전환 시점 세로 점선 + 패키지명 마커 (토글 가능)
  • hover → 해당 시점 모든 시리즈 스냅샷 표시
  • 각 시리즈는 자기 단위로 정규화 (Temp 25–85°C, Power 0–5W, GPU MHz 0–1GHz)

5. LLM 런타임 인스펙터

세션 기반 artifacts 수집

툴바 "● Start session" → artifacts/<timestamp>/ 폴더 자동 생성, 종료 시 report.md 자동 작성.

파일내용

realtime_metrics.jsonl1.5s마다 CPU/MEM/GPU/Temp/Freq/Power/thermal zones

Perfetto 자동 캡처

툴바 "Perfetto 30s" → 시스템 트레이스 (sched / cpu_freq / cpu_idle / binder / thermal / atrace gfx·view·wm·am·input) → perfetto_trace.pb pull. 브라우저에서 ui.perfetto.dev 에 드롭.

simpleperf 자동 record + report

프로세스 우클릭 → "simpleperf 30s record + report" → debuggable APK 한정으로 native 함수 단위 sampling → on-device simpleperf report 자동 실행 → .txt 결과 pull.

LLM 메트릭 인제스트

앱에서 한 줄만 찍으면 세션이 자동 수집:

Log.i("LLM_METRIC", JSONObject().apply {
    put("event", "done")
    put("model_name", "gemma-2b-it")
    put("backend_requested", "gpu")
    put("backend_actual", "gpu+cpu_fallback")
    put("ttft_ms", 612)
    put("tokens_per_sec", 34.8)
}.toString())

전체 스키마는 LLM_METRIC_SCHEMA.md 참고.

세션 비교 (before/after)

툴바 "Compare…" → artifacts 폴더 A/B 선택 → 두 세션의 realtime + LLM JSONL을 모아 평균·p95·Δ% 표 생성. 메트릭별로 "낮을수록 좋음"(CPU/MEM/GPU/Temp/Power/TTFT) vs "높을수록 좋음"(GPU MHz/tps) 방향을 반영해서 개선=초록 / 악화=빨강 / |Δ|<5%=회색 noise로 색칠. optimization 전후 빠른 회귀/개선 검증용.

병목 룰 엔진 (report.md 자동 생성)

세션 종료 시 JSONL 분석 → 스펙 §9 규칙 매칭:

  • thermal 전·후반기 평균 비교 → throttling 의심
  • GPU busy <20% + tokens/sec <5 → CPU fallback 의심
  • CPU peak >80% → CPU 포화 (simpleperf 권장)
  • TTFT >2000ms / decode <3 tok/s → 모델/quantization 검토

6. 디바이스 관리 / 자동 연결

  • 시작 시 adb start-server + adb devices -l 자동 호출
  • 툴바 콤보박스에 model + serial 표시, 디바이스 변경 시 forward/ping 자동 재시도
  • ↻ 새로고침 버튼으로 재스캔
  • 모든 adb 호출에 -s <serial> 자동 주입
  • 연결 직후 getenforce + id 자동 체크 → 로그에 "selinux=Enforcing (root needed for some power/sysfs reads)" 안내

12. 추가된 분석/자동화 기능 (이번 배치)

  • 자동 reconnect — stats/procs worker가 3회 연속 실패 시 adb forward+setup 자동 재시도 (in-flight guard)
  • Logcat 라이브 (메인 toolbar) — adb logcat 스트리밍 + level/tag/regex 필터 + Clear/Save, level별 색상
  • LLM live 차트 (메인 toolbar) — TTFT/tps/decode_ms 3-band 차트 + backend_actual 변화 dashed 마커 + thermal-aware tps (≥임계 °C 평균 따로 표시)
  • 통합 타임라인 마커 — 메인 차트 하단에 jank tick(빨강/진빨), 배경에 thermal throttle 빨간 띠 (히스테리시스 65→62°C). 토글 가능
  • Compare 윈도우 overlay 탭 — 두 세션을 같은 메트릭으로 A(빨강)/B(녹색) 라인 겹쳐 그리기
  • 자동 회귀 감지 — 세션 시작 후 30s baseline → 이후 ±3σ 이탈 시 로그 ⚠ 알림 (메트릭당 60s throttle, CPU/MEM/GPU/Temp/Power)
  • Battery breakdown (메인 toolbar) — dumpsys batterystats --checkin 파싱 → 추정 mAh per-uid, wakelock 표, raw 탭
  • Network I/O (per-UID) (우클릭 프로세스) — PID→UID 매핑 후 /proc/net/xt_qtaguid/stats(있으면 1Hz) 또는 dumpsys netstats detail(없으면 5s 폴링) 파싱. transports id를 wifi/cellular/vpn 등 라벨로 정규화. /proc/<pid>/net/dev는 net namespace 공유라 디바이스 전체 값이 나오는 문제 해결
  • 3D 뷰 확장 — 검색(class/resource-id/text/content-desc, Next 순회), Nav 버튼(↑parent ↓child ←prev next→), Measure 모드(A→B 중심거리/bbox overlap), Snap 버튼(현재 화면 썸네일)
  • Scenario runner (메인 toolbar) — 매크로 .sh + Perfetto/simpleperf 선택, N회 반복, iter간 sleep. 자동 세션 시작/종료 + 결과를 iter_NNN/ 서브폴더에 정리
  • Flame graph 뷰 (메인 toolbar) — simpleperf_report_*.txt의 callgraph 들여쓰기를 파싱해 Canvas flame graph. 클릭=zoom-in, 우클릭=zoom-out, ESC=reset, hover=함수명/percent
